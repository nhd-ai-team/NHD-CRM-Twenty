项目背景与目标

业务背景

新宏大集团上海分公司销售部主要面向海外市场，客户分布在中东、美洲、欧洲、东南亚、非洲等地区，属于外贸 B2B 业务模式。化工产品询盘周期较长、客户单价较高，客户关系管理质量直接影响订单转化与客户复购。

当前销售团队通过官网智能客服、WhatsApp、Facebook、Instagram、LinkedIn 等多个获客与沟通渠道触达客户；企业邮箱通过 IMAP/SMTP 接入 CRM，用于沉淀既有邮件往来记录。跨时区、多平台的业务模式对销售跟进和客户信息管理提出了较高要求。

项目目标

建立一套统一的客户关系管理系统，将多渠道客户线索、沟通记录、商机跟进集中沉淀，让销售可以在一个系统内完成从线索到成交的全流程管理，同时为管理层提供实时可查的业务数据。

核心价值：客户数据资产化，沟通记录不再散落，销售离职不带走客户，管理层实时掌握业务进度。

用户痛点

  • 1
    线索入口分散，没有统一管理

    询盘来自官网智能客服、WhatsApp、Facebook、Instagram、LinkedIn 等多个渠道，销售各自记录，无法汇总。线索容易被遗忘，也无法统计哪个渠道效果好。企业邮箱中的往来邮件通过 IMAP 同步归档，不作为主要获客渠道统计。

  • 2
    客户信息存在个人手里，离职即流失

    客户档案、沟通历史、报价记录分散在销售个人邮箱和聊天记录里，一旦销售离职，客户关系和跟进进度几乎归零。

  • 3
    跟进靠个人自觉，无节点提醒

    没有系统化的跟进节点管理，销售依赖自己记忆或个人便签，容易漏跟、晚跟。外贸时差大，错过回复窗口可能直接丢单。

  • 4
    多渠道沟通记录无法沉淀到客户档案

    WhatsApp 聊天记录、邮件往来、社媒私信分别在各自平台，销售需要在多个 App 之间切换才能了解客户全貌，接手客户时要重新了解背景。

  • 5
    管理层缺乏实时业务数据

    团队询盘量、商机转化率、各阶段客户数量等核心指标无法实时查看，需要靠销售手动汇报，数据滞后且不准确，难以快速发现问题。

  • 6
    官网获客无法自动进入业务流程

    官网访客咨询或留资后，需要人工手动录入 CRM，不仅效率低，也容易在时差窗口内漏掉真实意向客户。

3

验收标准

一期交付以下三个维度作为验收基准，三项均达标视为通过。

业务操作闭环
  • 一线销售能在系统内完整完成「线索录入 → 自动归集 → 自动分配 → 跟进记录 → 报价 → 商机推进 → 成交 → 合同归档」全流程
  • 无需依赖系统外 Excel 单独记录客户与商机信息
  • 多渠道线索（官网智能客服 / WhatsApp / 社媒）均有统一入口进入 CRM
管理数据可用
  • 管理层可通过仪表盘实时查看团队询盘量、各阶段转化率、待跟进客户数、业绩完成进度等核心指标
  • 数据与业务实际一致，无需等待销售手动汇报
  • 可按时间、销售、渠道等维度筛选查看
沟通自动沉淀
  • 邮件往来自动同步至对应客户档案，无需人工录入
  • WhatsApp / 社媒 IM 沟通记录通过 Chatwoot 接入，自动归档到对应客户
  • 官网智能客服会话结束后，客户信息与对话摘要自动写入 CRM

中间层核心能力要求

中间层是整个方案的稳定性关键，必须具备以下能力：

🔁

幂等去重

webhook 可能重复发送，中间层必须保证同一条消息只写入 CRM 一次

🔄

失败重试

网络波动写入失败时自动重试，死信队列记录失败日志，不漏同步

🔗

客户绑定

Chatwoot contact/conversation 与 Twenty contact 的稳定绑定关系维护

🤖

摘要生成

会话结束后调用 LLM 生成摘要、提取结构化信息，写入客户 Notes

🔒

安全隔离

Twenty API Key、Chatwoot Token 均由中间层保管，不暴露给官网前端

📦

字段映射

各渠道数据结构标准化，统一映射到 Twenty 字段，支持灵活配置

风险与注意事项

风险项 说明 应对建议
公网部署是前提 当前 CRM 运行在本地 localhost，官网、Chatwoot webhook、多渠道接入均无法连接 优先完成公网服务器部署，这是所有后续工作的硬前提
LinkedIn 接入不稳定 LinkedIn 私信 API 限制较多，不是 Chatwoot 的稳定原生渠道 单独做技术可行性验证，不作为一期 block 项；先用手动记录过渡
WhatsApp 规则限制 WhatsApp Business API 有 24 小时客服窗口规则，超时需使用消息模板 使用官方 Business API，不走个人号灰色方案；提前了解模板审核流程
webhook 重复写入 渠道 webhook 在网络异常时可能重复发送，导致 CRM 重复建档 中间层必须实现幂等逻辑，以 external_id 作为唯一键
网易邮箱限流 网易企业邮箱对第三方 IMAP 有严格频率限制，反复重连会触发封锁 现在邮件同步正常，不要随意断开重连；触发限流后等 15-30 分钟冷却
AI 客服信息抽取不完整 AI 从对话中抽取的客户信息可能不完整或有误 第一阶段保守设定：有邮箱或电话才写入；销售定期抽检 AI 写入的数据质量
CRM 数据膨胀 如果把完整聊天记录和附件全量存入 Twenty，会拖慢系统 CRM 只存摘要、标签和 Chatwoot 会话链接；完整记录保留在 Chatwoot
