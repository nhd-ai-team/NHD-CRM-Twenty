# 06 · 未来规划（官网 AI 客服对接）

最后更新：2026-07-01
状态：⬜ 规划中，尚未实施

---

## 目标

公司官网正在重建。计划让官网的 **AI 客服**与 Twenty CRM 打通，实现：

1. **自动获客**：访客跟 AI 客服聊天、留下联系方式后，自动在 CRM 建客户/公司/商机
2. **人工接管**：AI 答不上来的问题，转人工实时对话

---

## 整体架构

```
官网访客 ─→ AI客服(带LLM) ─→ 中间层 ─→ Twenty API ─→ CRM沉淀客户
                │                                        
                └─(答不上来)→ 转人工接管（在客服平台，不在CRM）
```

**三个 AI/角色，别混淆：**

| 角色 | 在哪 | 干什么 | 看得到官网对话 |
|---|---|---|---|
| AI 客服的 LLM | 官网/客服平台 | 对话 + 抽取信息 + 评意向度 | ✅ |
| 中间层 | 自己的服务器 | 格式转换（纯代码，不用 AI）| — |
| Twenty 的 AI | CRM 里 | 分析 CRM 已有数据 | ❌ |

---

## 需求一：访客留资 → 自动建客户（可行）

**流程：**
1. AI 客服的 LLM 一边对话，一边判断"要不要写入 CRM"（有没有有效联系方式、是不是真实意向）
2. 判断为是 → LLM 从对话抽取结构化信息（姓名/邮箱/电话/公司/国家/咨询产品/意向度）
3. 中间层接收，做字段映射
4. 中间层带 API Key 调 Twenty 接口写入客户/公司/商机
5. 客户几秒内出现在 CRM

**能自动获取的数据**：姓名、邮箱、电话、公司、国家、咨询产品、需求描述、意向度、来源、对话记录

**需人工后续补的**：客户真实性核实、商机金额/成交概率、客户背景调研、去重合并、报价合同附件

---

## 需求二：在 CRM 里跟访客实时聊天（❌ Twenty 做不到）

**重要现实**：Twenty 是 CRM，**不是在线客服系统**，没有实时聊天坐席功能。

**原因**：实时聊天需要 WebSocket（持续连接），而 CRM 的 API 是"一问一答"式的（REST/HTTP），不为实时对话设计。

**解决方案**：实时聊天 + 人工接管放在**专门的在线客服平台**做，Twenty 只做数据沉淀。
- 推荐：**Chatwoot**（开源可自托管，自带 AI + 人工坐席）、或 Crisp / Intercom（SaaS）
- 分工：聊天在客服平台，客户档案在 Twenty，两者 API 打通

---

## 需要开放的接口清单

```
官网/AI客服端：
  - 出站 Webhook（把客户数据推出来）        ← 必须
  - 实时消息 WebSocket（如要人工接管）      ← 看需求

中间层：
  - 一个接收地址（收官网推来的数据）        ← 必须
  - 调 Twenty API 的逻辑                    ← 必须
  - 保管 API Key（不能放官网前端）          ← 安全要求

Twenty 端：
  - REST/GraphQL API + API Key             ← 现成
  - Webhook（如要 CRM 反向通知）            ← 现成
  - 公网地址                                ← 需先部署到公网

实时聊天/人工接管（如做）：
  - 专门客服平台 + 坐席 API/WebSocket        ← Twenty 做不了，外接
```

---

## ⚠️ 前置条件（做这个之前必须先解决）

**Twenty 必须部署到有公网地址的服务器。**
现在跑在本地 localhost，官网（公网）连不到它。这是官网对接的**硬前提**。

建议：迁到一台 Linux 云服务器（阿里云/腾讯云/AWS 等），配好域名 + HTTPS + 防火墙。

---

## 分阶段落地建议

| 阶段 | 做什么 | 价值 |
|---|---|---|
| 阶段1 | 官网 AI 客服 + 自动留资进 CRM | 投入小、见效快，解决"漏接询盘、半夜没人" |
| 阶段2 | 人工接管 + 意向自动跟进 + Workflow 自动分配 | 提升响应质量和效率 |
| 阶段3 | 客户数据分析、营销自动化、复购经营 | 长期客户经营 |

---

## 价值与风险（决策参考）

**能解决的痛点**：询盘漏接、客户信息散落、海外时差错过商机、不知道谁值得优先跟、销售离职带走客户、官网获客效果看不清。

**优点**：获客自动化、24 小时不打烊（适合外贸跨时区）、客户数据资产化、响应快。

**风险**：AI 抓取不完美需人工抽检、脏数据需去重、过度依赖 AI 伤体验（人工通道必须畅通）、维护成本、隐私合规（GDPR）、初期投入。

**判断**：值得做，但分阶段。核心价值是"让官网从展示页变成 24 小时获客机器，每个客户沉淀成可经营资产"。最该警惕：别为全自动牺牲体验，AI 做接待+初筛，真人做深度跟进。
