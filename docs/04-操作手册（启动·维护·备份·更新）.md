## 一、日常启动 / 停止

```bash
# 启动（拉起所有容器）
docker compose up -d

# 停止（保留数据，只是停）
docker compose stop

# 查看运行状态
docker compose ps
```

启动后访问：http://localhost:3000

---

## 二、出问题时的排查

```bash
# 看网页服务日志
docker compose logs server --tail=50

# 看后台任务（邮件同步等）日志
docker compose logs worker --tail=50

# 网页打不开？先确认 OrbStack 在运行、容器都是 healthy
docker ps
```

**如果 Docker 命令报 "Cannot connect to Docker daemon"**：
→ 说明 OrbStack 没运行，打开 OrbStack 应用即可。

## 三、备份（重要！改动前必做）

```bash
# 1. 备份数据库（客户/公司/商机/邮箱配置都在这）
docker exec twenty-db-1 pg_dump -U postgres -Fc default > "backup/twenty_db_$(date +%Y%m%d_%H%M).dump"

# 2. 备份文件存储（附件等）
docker run --rm -v twenty_server-local-data:/data -v "$(pwd)/backup":/backup alpine tar czf /backup/server-local-data.tgz -C /data .

# 3. 确认备份生成
ls -lh backup/
```

> 备份文件在 `backup/` 目录。建议定期备份，并**把 backup 目录额外拷贝一份到别处**（网盘/移动硬盘），以防电脑故障。

---

## 四、恢复数据（从备份还原）

```bash
# 前提：db 容器在运行
# 1. 重建数据库
docker exec twenty-db-1 psql -U postgres -c 'DROP DATABASE IF EXISTS "default" WITH (FORCE);'
docker exec twenty-db-1 psql -U postgres -c 'CREATE DATABASE "default";'

# 2. 恢复（把文件名换成实际的备份文件）
docker exec -i twenty-db-1 pg_restore -U postgres -d default --no-owner < backup/twenty_db_20260629_1729.dump

# 3. 恢复文件存储
docker run --rm -v twenty_server-local-data:/data -v "$(pwd)/backup":/backup alpine sh -c "cd /data && tar xzf /backup/server-local-data.tgz"
```

## 五、升级 Twenty 到新版本

> ⚠️ **先备份！** 见上面第三节。开源项目新版可能有 bug，建议版本稳定后再升，别盲目追新。

```bash
# 1. 备份（务必先做）

# 2. 修改 docker-compose.yml 里的版本号（可选，默认 latest）
#    或直接拉最新
docker compose pull

# 3. 用新程序 + 旧数据重启（数据卷不动，数据保留）
docker compose up -d
#    首次启动会自动跑数据库迁移，把旧数据升级到新版结构
```

**怎么知道有新版**：去 GitHub 仓库 https://github.com/twentyhq/twenty
点 Watch → Releases only，有新版会邮件通知你。

## 六、彻底重装（会清空数据，慎用）

```bash
# ⚠️ 这会删除所有数据，重装前务必备份！
docker compose down -v    # 删容器+数据卷
docker compose up -d      # 全新空白重装
```
