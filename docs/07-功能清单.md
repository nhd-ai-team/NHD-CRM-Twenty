# 07 · Twenty 功能清单

最后更新：2026-07-01

> 说明：
> - 功能名称采用「中文界面名（英文原名）」形式，与 Twenty 实际界面一致。
> - 场景以外贸/国外客户业务为例。

---

## 一、核心数据（管什么）

| 功能名称 | 功能简要描述 | 应用场景 |
|---|---|---|
| 公司（Companies） | 以"公司"为单位管理客户单位，记录公司名、所在国家、网址、员工数等 | 把国外采购商按公司归档，一家公司下面挂多个联系人，看清是哪家企业 |
| 联系人（People） | 管理具体的人，记录姓名、邮箱、电话、职位，并关联所属公司 | 记录每个客户的采购、老板、跟单员，随时查邮箱电话 |
| 商机（Opportunities） | 管理每一笔潜在生意，记录金额、阶段、预计成交日、负责人、关联客户 | 跟踪每个询盘/订单：询价→报价→寄样→下单→付款→发货 |
| 任务（Tasks） | 待办事项，可设置截止日期、负责人，并关联到客户或商机 | 提醒"周五给沙特客户报价""下周跟进德国客户样品反馈" |
| 备注（Notes） | 记录沟通内容、要点、文档，关联到客户或商机 | 记下电话里谈的价格底线、客户的特殊要求，存档备查 |

## 二、查看与查找（怎么用起来）

| 功能名称 | 功能简要描述 | 应用场景 |
|---|---|---|
| 表格视图（Table View） | 像 Excel 一样按行列展示所有记录 | 一屏看全部客户，横向对比金额、阶段、国家 |
| 看板视图（Kanban / Board View） | 按阶段分列，卡片可拖拽移动 | 把商机按阶段摆开，客户付款了就把卡片拖到"已付款" |
| 日历视图（Calendar View） | 按日期展示记录 | 按预计成交日/跟进日排布，看时间安排 |
| 过滤（Filter） | 按任意字段组合筛选记录 | 一键筛出"中东 + 高意向 + 本月未跟进"的客户 |
| 排序（Sort） | 按字段升/降序排列 | 按金额从高到低排商机，优先跟大单 |
| 视图（Views） | 把筛选/排序条件保存成一个视图，下次一键打开 | 存一个"我的高意向客户"视图，每天点开就是筛好的 |
| 搜索（Search） | 全局搜索客户、公司、邮箱等 | 输入公司名或邮箱秒找到对应客户 |

## 三、邮件与日历

| 功能名称 | 功能简要描述 | 应用场景 |
|---|---|---|
| 账户 / 电子邮件（Accounts / Emails） | 接入邮箱（IMAP/SMTP），把往来邮件自动归档到对应客户 | 跟客户来回的报价邮件自动出现在该客户页面，不用翻 Outlook |
| 日历（Calendars） | 接入日历（CalDAV），同步会议/日程到客户 | 跟客户约的视频会议，自动关联到客户记录 |
| 拦截列表（Blocklist） | 设置不同步的邮箱/域名 | 排除内部同事、垃圾域名的邮件，不进 CRM |

> ⚠️ 注意：Twenty **不能在界面直接写邮件/群发**，回复会跳回你的邮箱客户端。

## 四、自动化与 AI

| 功能名称 | 功能简要描述 | 应用场景 |
|---|---|---|
| 工作流（Workflows） | 设置"触发条件 + 自动动作"，让系统自动干活 | 客户进入"已付款"阶段，自动建发货任务、自动通知负责人 |
| 工作流运行记录（Workflow Runs） | 查看每次自动化的执行记录 | 检查"自动跟进邮件"有没有正常发出 |
| 工作流版本（Workflow Versions） | 管理工作流的不同版本 | 改了自动化规则后，保留旧版本可回溯 |
| AI 助手（Ask AI） | 用大白话问 CRM 里的数据 | 问"这个月哪些客户最值得优先跟进" |
| AI 智能体（AI Agent，工作流内） | 在自动化流程中让 AI 介入处理 | 让 AI 自动读邮件内容、判断意向度、填写字段 |

> ⚠️ 注意：AI 功能较新，部分能力仍在完善；Twenty 的 AI 只处理 CRM 内已有数据。

## 五、分析与展示

| 功能名称 | 功能简要描述 | 应用场景 |
|---|---|---|
| 仪表盘（Dashboards） | 自定义数据看板，展示统计图表 | 看业绩总额、客户国家分布、销售漏斗转化 |
| iFrame 小部件（iFrame Widget） | 在仪表盘里嵌入外部网页 | 把公司生产计划看板、外部报表嵌进来一起看 |

## 六、协作与管理

| 功能名称 | 功能简要描述 | 应用场景 |
|---|---|---|
| 成员（Members） | 邀请同事、分配账号 | 给每个业务员开账号，一起用同一套客户库 |
| 权限 / 角色（Roles & Permissions） | 控制不同成员能看/能改哪些数据 | 限制业务员只能看自己的客户，主管能看全部 |
| 数据模型（Data model） | 自定义对象、字段、字段类型 | 加"客户来源""意向度""所属洲"等外贸专用字段 |
| 布局（Layout） | 自定义记录页面的排版、导航 | 把客户页面最常用的字段放最上面 |
| 常规（General） | 工作区基础设置（名称、外观等） | 设置公司工作区名称、Logo |
| 个人资料（Profile） | 个人账号信息设置 | 改自己的名字、头像、密码 |
| 体验（Experience） | 界面语言、主题、时区、日期格式 | 切换中文、设置为北京时间 |

---

## 七、数据与集成

| 功能名称 | 功能简要描述 | 应用场景 |
|---|---|---|
| 导入（Import） | 用 CSV 批量导入记录 | 把手上的 Excel 客户名单一次性导进来 |
| 导出（Export） | 把记录导出成文件 | 导出客户名单做备份或给别的系统用 |
| API 和 Webhooks（APIs & Webhooks） | 生成 API Key，供外部系统读写数据 | 让公司官网 AI 客服自动把访客写进 CRM |
| 应用（Apps） | 安装/扩展第三方应用能力 | 扩展 Twenty 功能 |
| 管理面板（Admin Panel） | 管理员级的系统配置（含 AI 提供商配置） | 配置 OpenRouter 等 AI 模型 |

---

## 附：Twenty 明确做不到的（避免误用）

| 事项 | 说明 |
|---|---|
| 界面直接群发邮件 | 需外接邮件营销工具（Mailchimp/SendGrid 等） |
| 实时在线客服 / 跟网站访客聊天 | Twenty 是 CRM 不是聊天系统，需外接 Chatwoot 等 |
| 营销获客 / 线索挖掘 | 不自带，需外部工具 |
| 客户服务工单 | 需用"数据模型"自建对象 |
